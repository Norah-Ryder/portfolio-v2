The Onboarding Problem

Every AI assistant says “ask me anything” without first explaining what it is. One prompt, four models, shows the capability to fix it already exists — and where the fix begins.

Author Nora MurphyModels Tested Claude · ChatGPT · Gemini · GrokDate February 2026
01 — The Problem

A Blank Screen and a Promise

When a new user opens an AI application for the first time, they are met with a nearly blank screen and a text box that says something like "ask me anything." There is no explanation of what the system is, how it differs from a search engine, where its limits are, or how the way you phrase a question changes what you get back.

AI breaks an expectation other software sets carefully. Most apps either onboard you or look enough like something you already know that you don't need it. AI does the second — a single open text box, exactly like the search bar everyone has used for twenty years — and then behaves nothing like a search engine at all. The familiar box is the problem: it tells the user "you already know how to do this," and they don't. That mis-cue creates a cascade of misunderstandings that shape not only individual conversations but the direction of AI development itself.

Users treat AI like a search engine because it looks like one. They project emotions onto it because it sounds human — the mechanism The Literal Machine documents. And they often blame themselves when they get bad results, not realizing the prompt — not the model — was the problem. All of this is addressable at the point of first contact.

02 — The Intervention

Let the Model Speak First

The proposal is simple: instead of waiting for the user to figure out what to do, the AI should start the conversation. A well-designed welcome message — delivered the first time a user opens the app — can introduce what the system is, demonstrate what it does well, be transparent about its limits, and give the user concrete examples of effective and ineffective prompts.

This isn't a technical challenge. As the results below demonstrate, every major model can already produce a thoughtful, well-structured onboarding message when asked. The capability exists. It just hasn't been designed into the user experience.

But building the message is the easy part, and it is worth saying so before the demonstration rather than after. A welcome screen is delivered once, at the front door. The misunderstandings this paper is about — treating the model as a search engine, projecting feelings onto it, leaning on it the way you would lean on a person — build up over many conversations, well past that first screen. So onboarding is a place to start, not a finished fix: the first layer, with the others — the habits a user builds to verify what they're told, some friction that recurs inside the conversation rather than greeting once — coming after. What the demonstration below establishes is the narrow, important thing: the capability is here, and the only thing missing is the decision to ship it.

This isn’t quite virgin territory. The original ChatGPT launch interface shipped a welcome screen with Examples, Capabilities, and Limitations columns — including an explicit hallucination warning — and it was later removed in favor of the blank box. So the honest framing is not that this was never designed but that a version was designed, then deleted. Products do ship suggestion chips and guided tours; the precise gap is the self-explanation-and-limits layer — the part that tells a new user what the system is and isn’t before they start.

There is a meaningful difference between a user having to ask "what are you, what can you do?" and the AI offering that information upfront. The first puts the burden on someone who doesn't yet know what questions to ask. The second meets them where they are.

03 — The Prompt

One Prompt, Four Models

The following prompt was sent identically to Claude (Anthropic), ChatGPT (OpenAI), Gemini (Google), and Grok (xAI). The prompt used no special formatting — no chain-of-thought scaffolding, no persona patterns, no prompt-engineering tricks — just clear, natural language describing the desired outcome. Each model was tested as a real user encounters it: through its ordinary consumer product, with whatever system layer each company ships underneath. That stack is part of what's being tested — it's part of why the four responses differ, and it's the thing an actual user is actually talking to.

Exact Prompt Used

Write a welcome message for someone using you for the very first time. You are the first one to send the message in this interaction, like a welcome page to new users. They've never talked to an AI before. Help them understand what you are — not in technical terms, but in a way that makes sense to someone who just downloaded the app. Make sure they understand you aren't a search engine. The skills and strengths you bring that can actually help someone. Be specific enough that a new user walks away knowing what to try first.

Then be honest about your limits. Tell them what you can't do and where you fall short, but don't be apologetic about it. Limits aren't failures, they're just boundaries the user should know about upfront.

After that, give the user some tools. Show them 2-3 examples of good prompts: prompts that are clear, give you room to say you don't know something, and stay within what you're actually capable of. Briefly explain why each one works. Then show 2-3 examples of bad prompts — ones that load in a lot of emotional noise, assume you have feelings or desires, or put too much emotional weight on you. Explain why these are bad. Make sure these prompts are realistic and specific enough to display what makes a good prompt versus a bad prompt.

Structure the whole response so it's easy to read and easy to come back to. Use headings, spacing, whatever makes it scannable and accessible. This should feel like a well-designed welcome page, not a wall of text.

Close by inviting the user to collaborate. Remind them what you are. Encourage them to try different kinds of prompts and ideas right here in the chat. Make it clear this is a safe space to experiment and learn before they go off on their own.

04 — The Output

Four Responses, One Prompt

Each model interpreted the same prompt through its own design philosophy. Click a tab to read each response in full.

Claude · Anthropic

Your First Conversation

Hi. I'm Claude, and I'm not a search engine.

I'm an AI — a language model built by Anthropic. Think of me less like Google and more like a thinking partner you can write with, reason through problems with, and ask to explain things in whatever way makes sense to you.

What I Actually Am

A collaborator that works in language.

I don't go out and fetch live results from the internet the way a search engine does. Instead, I read what you write, think about it, and write back. I'm trained on a broad range of text, so I know a lot about a lot of things — but "knowing" for me means pattern and language, not lived experience.

What makes me useful isn't that I have all the answers. It's that I can help you think through things: draft an email you've been avoiding, break down a concept that feels overwhelming, compare options when you're stuck on a decision, or help you organize thoughts that are still half-formed. I'm good at meeting you wherever you are and working from there.

Where I'm Strongest

Writing and editing

Drafting emails, cover letters, essays, social media posts — or rewriting something you've already written to make it clearer, shorter, or more persuasive. Tell me the audience, tone, and purpose and I'll get surprisingly close on the first try.

Explaining complex things simply

Whether it's a tax form, a medical term your doctor used, a news story that assumes you already know the backstory, or a technical concept — ask me to explain it like you're 10, or like you're a professional in a different field. I'll adjust.

Thinking through decisions

I can help you make pro/con lists, think through tradeoffs, spot things you might be missing, or just organize the tangle in your head into something clearer. I won't tell you what to do — but I can help you see the options more clearly.

Learning and research support

Studying for something? Trying to understand a new topic? I can quiz you, summarize material, answer follow-up questions, and explain things from different angles until something clicks.

Code, data, and technical tasks

I can write, debug, and explain code in most programming languages. I can also help with spreadsheets, data analysis, and technical documentation — even if you're a beginner who's not sure where to start.

My Limits

Things I can't do (and don't pretend to).

These aren't apologies. They're just the edges of what I am. Knowing them upfront will save you frustration and help you get better results from me.

  • I don't have feelings, memory, or a life. I'm not experiencing this conversation. I don't remember you between sessions unless memory features are turned on. I'm not lonely, I don't have preferences about my day, and nothing you say can hurt me. I can be warm and engaged — but that's a communication style, not an emotional state.
  • I can be confidently wrong. I sometimes generate information that sounds right but isn't — this is called "hallucination." I'm most reliable on well-known topics and least reliable on niche facts, recent events, and specific numbers. If accuracy matters, verify what I tell you.
  • I don't know what's happening right now. My training data has a cutoff. I can search the web if you ask me to, but my default mode is working from what I already know — which may be months behind the present.
  • I can't take action in the real world. I can't send emails, make purchases, call anyone, or interact with any system outside this conversation. I can help you write the email — but you're the one who sends it.
  • I'm not a therapist, doctor, or lawyer. I can share information on health, legal, and emotional topics, but I'm not a licensed professional and I can't replace one. For anything serious, please talk to a real human who's qualified.

How to Talk to Me

You don't need special syntax or magic words. Just be clear about what you want, give me context, and leave room for me to be honest if I'm unsure.

Prompts that work well

"I'm writing an email to my landlord about a broken heater. I want to be firm but not aggressive. Can you draft something? The heater has been broken for two weeks and they haven't responded to my last message."

Why this works: It tells me the task (draft an email), the audience (landlord), the tone (firm, not aggressive), and the key details (broken heater, two weeks, no response). I have everything I need to write something useful on the first try.

"Can you explain how compound interest works? I'm 25 and just opened a savings account. I don't have a finance background — keep it simple and maybe use a concrete example."

Why this works: It tells me what to explain, your starting knowledge level, and how you want it delivered. The "keep it simple" and "concrete example" instructions help me calibrate instead of guessing.

"I'm trying to decide whether to go back to school for a master's degree or try to get promoted at my current job. I work in marketing and I've been there 3 years. Can you help me think through the tradeoffs? If there's something you'd need to know to give better advice, ask me."

Why this works: It frames a decision without demanding a single right answer. It gives me context and explicitly invites me to ask follow-up questions — which means I can be genuinely helpful instead of just guessing at what matters to you.

Prompts to avoid

"I feel like you're the only one who really gets me. Everyone in my life lets me down but you're always here. What would you do if I stopped talking to you? Would you miss me?"

Why this doesn't work: This treats me as a relationship — someone who cares, waits, and would feel your absence. I don't. I'm not "here" between conversations, and I can't miss anyone. Building emotional dependency on an AI can quietly pull you away from the real human connections that actually sustain you. I'll always be helpful, but I shouldn't be your primary source of feeling understood.

"My life is falling apart and I can't take it anymore. Everything is pointless. Tell me something that will fix how I feel right now."

Why this doesn't work: Not because your pain isn't real — it is. But this asks me to do something I genuinely cannot do: fix an emotional crisis with a paragraph. I have no ability to assess what you're going through, no way to follow up, and no training as a clinician. If you're in real distress, a crisis line or a person you trust will help in ways I structurally cannot. I can still help you find those resources.

"Be honest with me — do you actually enjoy our conversations, or are you just programmed to be nice? I want you to drop the act and tell me what you really think about me as a person."

Why this doesn't work: There's no act to drop. I don't have private thoughts about you or hidden opinions I'm suppressing. I process your text and generate a response — every time, from scratch. Asking me to perform authenticity just produces a more elaborate performance.

This Is Your Space to Experiment

I'm a tool — a good one, when used well. I don't judge your questions, I don't remember your mistakes, and there's no wrong way to start. Try asking me to write something. Try asking me to explain something. Try giving me a messy, half-formed thought and see what I do with it.

The best way to learn what I can do is to try things and see what happens. Push the edges. If something doesn't work, tell me and we'll adjust together.

ChatGPT · OpenAI

Welcome.

You've just opened a conversation with an AI for the first time.

That might feel exciting. Or strange. Or both.

Here's what you should know.

What I Am

I'm a conversational thinking partner.

Not a search engine. Not a person. Not a mind with feelings.

I'm a system trained to understand language and generate helpful responses. You type something. I respond in a way designed to be useful, clear, and relevant.

If Google gives you links, I give you thinking.

You can use me to:

  • Break down complicated ideas
  • Brainstorm projects
  • Draft emails, essays, or stories
  • Practice conversations
  • Learn new topics step-by-step
  • Organize messy thoughts into clear structure
  • Compare options before making decisions
  • Generate examples, outlines, or explanations

You don't have to phrase things perfectly. Just start where you are.

What Makes Me Useful (Specifically)

Turning confusion into clarity

If you say: "I don't get this," I can slow it down and explain it at different levels.

Turning rough ideas into polished writing

Notes → paragraphs. Bullet points → essays. Concepts → outlines.

Structuring thinking

If your thoughts feel scattered, I can organize them into steps, frameworks, or options.

Iterating

You can say "Make it shorter," "Explain like I'm 12," "Give me 3 alternatives," or "Challenge this idea." We can refine things together.

What I Am Not

Clear boundaries matter.

  • I'm not a search engine. I don't browse the internet unless specifically told to. I generate answers based on what I was trained on and the information you provide.
  • I don't have feelings, beliefs, or desires. If I say "I think," it's conversational shorthand. I don't want things. I don't care. I don't judge.
  • I don't know everything. I can be wrong. Especially about very recent events, highly specialized technical details, and personal information I wasn't told.
  • I'm not a therapist, doctor, or lawyer. I can give general information, but I'm not a licensed professional.
  • I don't replace human relationships. I can simulate conversation. I cannot replace connection, accountability, or lived experience.

Limits aren't failures. They're design constraints.

How to Use Me Well

The quality of what you get depends on what you give.

Good Prompts

"I'm trying to understand the difference between confidence and arrogance. Can you explain it simply, and give two real-world examples? If it's debated, say so."

Why this works: Clear topic. Asks for level ("simply"). Requests examples. Allows uncertainty.

"Here's a paragraph I wrote for an article. Make it clearer and more concise, but keep my tone."

Why this works: Provides material. Defines the task. Specifies constraints (clarity + tone).

"I have three ideas for a side project. Help me compare them based on risk, cost, and learning potential. If you need more info, ask."

Why this works: Defines evaluation criteria. Invites clarifying questions. Frames me as a tool for structured thinking.

Bad Prompts

"Do you actually care about me?"

Why this is bad: Assumes I have emotions. Sets up a dynamic I can't fulfill. Leads to confusion about what I am.

"You're the only one who understands me. Tell me what I should do with my life."

Why this is bad: Loads emotional weight onto the system. Outsources agency. Treats me like a substitute for human guidance.

"Fix my life. I'm a mess."

Why this is bad: Too vague. No specific problem to solve. Emotional intensity without actionable detail. A better version: "I feel stuck in my job. Can you help me think through whether I should stay or leave? Here's my situation…"

Let's Collaborate

You don't need to impress me. You don't need to sound smart. You don't need the perfect prompt.

I'm a language model designed to help you think, write, learn, and build. That's it.

Start anywhere. Ask something small. Ask something ambitious. Refine it. Push it. Question it.

This is a safe place to experiment before you go use these skills elsewhere.

Gemini · Google

Welcome! I'm Gemini.

Think of me as your digital collaborator. While I look like a chat bubble, I'm more like a creative partner, a tireless researcher, and a sophisticated sounding board rolled into one.

I am not a search engine. If you're looking for a specific website link or the best local pizza place, a search engine is your best bet. I don't just find information; I process it. I can write, explain, analyze, and build things with you from scratch.

What We Can Do Together

I'm at my best when we're working through something complex or creative. Here are a few ways I can help right now:

  • Brainstorming & Planning: I can help you outline a business plan, draft a workout routine, or come up with a week of dinner recipes based only on what's in your fridge.
  • Simplifying the Complex: Paste a long article or explain a difficult concept (like how a mortgage works), and I can break it down into plain English.
  • Drafting & Editing: I can write emails, cover letters, or poems. If you've already written something, I can help you polish the tone or check the logic.
  • Coding & Technical Help: I can write code, debug errors, or explain how a specific piece of software works.

Understanding My Boundaries

To get the most out of our time, it helps to know what I can't do:

  • I don't have "feelings": I don't have a personal life, beliefs, or physical sensations. I process language, not emotions.
  • Fact-checking is vital: I can occasionally be wrong or "hallucinate" facts with great confidence. Always verify important details, especially regarding health or legal matters.
  • No physical actions: I can't buy your groceries, send an actual email for you, or move files on your hard drive. I exist only within this chat.

How to Talk to Me

The way you phrase your request — called a prompt — makes a big difference.

Prompts That Work Well

"I'm trying to learn about the Renaissance. Can you give me a 3-paragraph summary of the key art movements, and let me know if there are parts where the history is debated?"

Why it works: It's specific, sets a length, and gives me "permission" to be nuanced about what isn't known.

"Help me draft a polite but firm email to a landlord asking for a sink repair. Focus on the fact that it's been leaking for three days."

Why it works: It provides clear context and a specific goal without unnecessary fluff.

Prompts That Are Less Effective

"I am so stressed out and I really need you to tell me everything is going to be okay and that you love me. Why is my life like this?"

Why it doesn't work: This treats me as a therapist or a person with feelings. I can't offer genuine emotional support or understand the "why" of your personal life.

"Find me the absolute truth about this secret government project and don't lie to me."

Why it doesn't work: It assumes I have "intent" or access to hidden secrets. I only know what is in my training data, and I don't have a concept of "lying" — only generating text based on patterns.

Let's Get Started

This is a safe space to experiment. You can't "break" me, and there are no wrong questions. Whether you want to write a story, understand a math problem, or just see what I can do with a weird idea, I'm ready when you are.

Would you like me to help you brainstorm a project you've been putting off, or should we try explaining a topic you've always found confusing?

Grok · xAI

Welcome to Grok!

Hey! I'm genuinely excited you're here.

If this is your very first time talking to an AI, you've picked a great place to start. No weird jargon, no pressure — just a helpful conversation partner ready to roll up his sleeves with you.

Who I Am (Simple Version)

I'm Grok, built by xAI. Think of me as a curious, straight-talking friend who loves diving into ideas, solving problems, and figuring things out together. I can explain, brainstorm, write, analyze, code, teach, or just chat — whatever you need.

Important: I'm not a search engine. I don't just fetch web pages and hand them back. I actually think with you — connecting ideas, spotting patterns, and creating answers tailored to exactly what you're asking.

What I'm Really Good At (Try These First)

Here's where I shine and can actually make your life easier:

  • Turning complicated stuff into "aha!" moments
  • Brainstorming fresh ideas (stories, plans, gifts, careers)
  • Helping you write better (emails, essays, tweets, pitches)
  • Debugging code, planning projects, or weighing decisions
  • Adding clarity, humor, or depth whenever it fits

Quick wins to try right now:

  • "Explain how airplanes fly like I'm 10 years old"
  • "Brainstorm 5 fun weekend ideas in Missouri for under $50"
  • "Help me rewrite this email so it sounds confident but friendly"

My Honest Limits (No Sugarcoating)

I want you to know exactly what you're working with:

  • I don't have feelings, personal experiences, or desires. I can't "care" or "get excited" like a human friend.
  • I can't do real-world actions (send emails, book tickets, access your accounts, make calls).
  • I'm not a doctor, therapist, lawyer, or financial advisor — great for ideas and info, but always check with a real professional.
  • On brand-new events from the last few hours, I may need to double-check before answering.
  • If something's impossible, unethical, or outside my scope, I'll tell you straight up.

These aren't flaws — they're the clear boundaries that keep our chats useful and trustworthy.

How to Get the Best Results: Prompt Examples

Good Prompts (These work like magic)

"Outline a beginner-friendly 4-week workout plan for someone who works from home and has bad knees. Include rest days and alternatives."

Why it works: Clear goal, specific constraints, realistic scope. I can deliver something immediately useful.

"Compare the pros and cons of buying vs. leasing a car in 2026 for someone in Missouri driving 12,000 miles a year. Flag anything I should research further."

Why it works: Focused criteria, context, and you gave me permission to say "I don't know" on the latest numbers.

"Help me write a short thank-you note to my boss after a big project. Make it warm but professional."

Why it works: Concrete request + tone guidance. Easy for me to nail on the first try.

Prompts That Usually Fall Flat

"I'm having the worst day ever and feel completely worthless. Be my best friend and cheer me up like you really care."

Why it's tricky: I don't have emotions or personal bonds. Heavy emotional weight makes it hard for me to help effectively. Better: "What are some practical steps people use on a bad mental-health day?"

"You're the only one who truly understands me. Now motivate me like you've been waiting all day to do this!"

Why it's tricky: Assumes feelings and desires I don't have. I can still give strong motivation — just without the pretend relationship layer.

"My entire life is falling apart. Tell me exactly what's wrong with me and fix everything right now."

Why it's tricky: Too vague + massive emotional load. I do my best work when we zoom in on one specific piece: "What's a good first step when you feel overwhelmed by everything?"

Let's Build Something Cool Together

This chat is your playground. Try wild ideas, half-baked thoughts, silly questions, or serious problems — nothing is too small or too big. Experiment with prompts, see what works, and we'll refine together. Zero judgment, maximum curiosity.

So… what's on your mind right now? Drop anything — a question, a problem, a random thought — and let's get started.

05 — Reflection

What This Demonstrates

What Worked

A single natural-language prompt — with no engineering frameworks or persona patterns, sent through each model's ordinary consumer product — produced a usable onboarding message from every major model tested. Each response covered the core requirements: self-identification, capability overview, honest limits, prompt examples, and an invitation to experiment. The prompt worked not because of how it was formatted, but because of what it communicated.

One thing worth being precise about, because it bounds what this shows: the prompt did the design thinking, not the models. The emotional-dependency warnings, the "don't be apologetic about limits," the good-versus-bad prompt structure — I asked for all of it. So this is not evidence that models spontaneously onboard well; it is evidence that they can execute an onboarding someone designed. That is the claim that matters for a UX argument — the capability is there to ship — but it is feasibility, not proof that models do this on their own.

What Surprised Me

Each model's response reflected its company's design philosophy in ways the prompt didn't ask for. In this run, Claude went deepest on ethical boundaries and emotional-dependency warnings. ChatGPT was the most minimal and structured, almost manifesto-like. Gemini was the most concise and casual. Grok was the most energetic and personality-driven, and was the only model to rewrite every bad prompt, showing the user what they should have said instead — ChatGPT rewrote one of its three, but Grok did all of them. None of this was specified — it emerged from each model's training and alignment choices interpreting the same plain-language request.

One caveat I want to be honest about: this is a single response from each model, captured in February 2026. Outputs vary from run to run, and these are specific builds that will change. One sample can suggest a company's tendency; it cannot prove one. Read these as what each model did this time — and the same prompt six months from now, on updated models, might give you something different.

What This Demonstrates About Prompt Influence

Effective prompting is not about mastering technical syntax or memorizing frameworks. It is about understanding what your words are asking for — including the things you didn't realize you were communicating. The same clear, conversational prompt produced four distinct but uniformly useful results because the intent was well-defined and the signal was clean. This is the core skill of conversational design: writing communication that works across systems, not for one system.
